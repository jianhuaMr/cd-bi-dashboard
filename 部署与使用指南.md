# CD-BI 店铺清算分析系统 · 部署与使用指南

## 这是什么

一个类 Power BI 的轻量数据可视化系统:每月把各店铺 정산리스트(清算表)Excel 原件拖进去,自动识别店铺与月份、自动校验数据、生成全套交互看板。

**核心特性**
- 纯静态网页,无后端 —— 清算数据**不上传任何服务器**,只保存在使用者自己的浏览器(IndexedDB)
- 自动识别 5 家店铺:PDD-Deepte / PDD-SSF / PDD-CD BROWN / 鲸芽(징야) / 快手-HOKABOMB(호카밤)
- 表头动态识别列结构(各店列布局不同、未来小幅变动也能适配),失败时回退到预置映射
- 上传时自动执行 4 项完整性校验(KRW换算 / 利润公式回算 / SKU加总 / 库存加总),异常会黄牌提示
- 模块:集团总览(KPI+店铺对比+智能诊断) / 历史趋势 / 店铺明细(P&L瀑布+费用构成) / SKU·ABC帕累托 / 库存预警(5级分级) / 快手店快照 / 数据中心(备份/恢复)

## 部署方式(三选一)

### 方式 A:Netlify Drop(最简单,约 2 分钟)
1. 打开 https://app.netlify.com/drop (需注册免费账号,支持邮箱/GitHub注册)
2. 把整个 `bi-dashboard` 文件夹拖进页面
3. 得到一个 `https://xxxx.netlify.app` 网址,即可分享给同事
4. 可在 Site settings → Domain 里改成好记的前缀

### 方式 B:Vercel
1. 注册 https://vercel.com (免费)
2. 安装 Vercel CLI 或直接在网页 New Project → 上传 `bi-dashboard` 文件夹
3. 部署完成得到 `https://xxxx.vercel.app`

### 方式 C:GitHub Pages
1. 新建一个 **Private 不行、需 Public** 的仓库(或付费私有Pages),把 `bi-dashboard` 内容推上去
2. Settings → Pages → Source 选 main 分支根目录
3. 访问 `https://<用户名>.github.io/<仓库名>/`

> 不想注册任何账号?直接双击 `index.html` 也能在本机使用(需联网加载图表库)。

## 每月使用流程

1. 打开网址 → 输入密码
2. 进入「数据中心」→ 把当月各店清算表 Excel 一次性全部拖入
3. 查看上传日志:绿色=成功+校验通过;黄色=成功但有校验警告(按提示人工复核);红色=解析失败
4. 切到「集团总览」即可查看最新月份全套分析

**注意事项**
- 同一店铺同一月重复上传会自动覆盖(以最后一次为准),传错文件重传即可
- 每次上传会把该文件 summary 里的**全部历史月份**一并入库,所以即使只传最新月文件,历史趋势也是完整的;但 SKU/库存明细只有该文件对应月份的
- 数据存在浏览器里:**换电脑/换浏览器/清除浏览数据前,务必先「导出备份 JSON」**,在新设备「导入备份」即可
- 给同事共享数据:导出备份 JSON 发给对方,对方在自己浏览器导入
