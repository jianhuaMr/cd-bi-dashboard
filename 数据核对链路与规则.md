# CD-BI 数据核对链路与规则

## 一、数据来源(6类上传)

| # | 上传文件 | 进入的库 | 提供什么 |
|---|---|---|---|
| 1 | 各店**清算表 정산리스트** | `summary`(月度P&L历史) + `detail`(SKU피벗/재고현황) | 各店 GMV/退款/费用/净销售/利润/汇率 + SKU与库存明细 |
| 2 | **成本分析表 매출대비 원가분석** | `monthlyCostAnalysis` | **全渠道月度权威源**:每渠道 매출액/매입액/이익금/이익율(含快手、清潭B2B) |
| 3 | **月末库存表 PMP0410** | `inventoryMonthly` | 各仓 품번 期初/入库/出库/期末件数 |
| 4 | **产品信息表 SYC1110** | `productMaster` | 품번 采购价/标准成本/单位/名称(产品登记) |
| 5 | **汇率**(手录 / 一键回填) | `exchangeRates` | 每月 USD→KRW / CNY→KRW 官方汇率 |
| 6 | **快手清算** | `kuaishou` | 快手历次清算(不规律结算) |

## 二、SSOT 单一可信源原则
- 同一指标**只在数据层取整一次**(`canon`,KRW 用整数 won),所有展示位直接读取,**禁止各组件再 round / 再浮点求和**。
- 金额全程整数最小单位(KRW=won;USD/CNY=1/10000),换算用 BigInt,只在展示落小数。

## 三、月度合计(总览 TTL)取数优先级
1. **当月有成本分析表** → 以它为准,全渠道(拼多多×3 + 鲸芽 + 快手 + 清潭),与会计「월 합계」一致。
2. **当月无成本分析表** → 回退清算表4店 + 已结算快手。
- 快手:有清算(出现在成本分析/清算)就计入;没有=当月0。

## 四、核对链路(逐项规则)

**A. P&L 闭合(店铺级)**
`净销售KRW − 出库原价KRW + 费用KRW(负,已含广告/技术/CS/其他收入) = 营业利润KRW`
- 容差 2 won;首尾不等 → UI 红字报警。

**B. 内部标准核对(以成本分析表为标准答案)**
- 逐店比对:清算表的 **净销售** 与 **出库原价** 应 = 成本分析表对应渠道的 매출액 / 매입액(容差 100won 或 0.5%)。
- **不比营业利润**:成本分析 이익 = 매출−매입(毛利),清算营业利润还要扣平台费,口径不同。
- 不比库存账(口径不同)。
- B2B清潭、快手(无清算明细)不参与核对。

**C. 汇率校验**
- 「清算成交汇率」= 清算文件隐含(净销售KRW ÷ 净销售USD),独立于手录官方汇率。
- 偏差 = 成交 vs 官方,>0.5% 标黄;两者完全一致显示「⚠同源」(提示改录独立官方价)。
- 官方汇率环比突变 >3% 高亮待复核。

**D. 库存成本匹配优先级**
本品采购价>0 → 本品标准成本>0 → **基础品번继承**(FOC/-2变体取主品成本)→ 库存表自带单价 → 期末金额反推 → 都无则标「缺少采购单价」(金额KPI排除)。

**E. SKU 销售联动 QA**
- 库存表「出库」 vs 店铺结算「SKU销量」按 품번对比,差异>10%且≥2件列出(口径不同,小幅差异正常)。

**F. 比率口径**
- 退款率 = |退款USD| ÷ GMV USD(卖出额口径,可能跨期)。
- 费用率 = |费用USD| ÷ 净销售USD。
- 周转天数 = 期末件数 ÷ (近3月月均出库/天);月均出库<10件 → 「不可估」。

**G. 上传完整性自检**(清算表)
KRW换算、利润公式回算、SKU加总vs净销售、库存加总vs期末库存 — 异常黄牌。

## 五、防数据丢失
- 所有数据仅存浏览器 IndexedDB(按网址+浏览器隔离)。
- **定期「导出备份 JSON」**;换设备/清缓存前必须先导出,新设备「导入备份」恢复(含汇率)。
